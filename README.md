# PrivateSEO Аналитика

Open-source Telegram-бот, который превращает данные Яндекс Метрики в короткий ответ:

1. что изменилось;
2. какой источник или страница повлияли;
3. что стоит проверить.

Штатный Telegram-отчёт Метрики перечисляет показатели. Этот бот сравнивает два одинаковых периода, отсеивает шум малых чисел, находит основной источник и посадочную страницу изменения и предлагает следующее действие.

## Просто пользоваться — без установки

**[Открыть PrivateSEO Аналитику в Telegram →](https://t.me/private_seo_analytics_bot)**

Ничего скачивать, настраивать или размещать на сервере не потребуется: нажмите Start, подключите Метрику и выберите сайт.

Репозиторий открыт для прозрачности и независимости: можно проверить, как бот работает с данными, предложить улучшение или развернуть собственную копию. Большинству пользователей это не понадобится.

## Что уже работает

- OAuth 2.0 Authorization Code + PKCE, доступ только `metrika:read`;
- зашифрованное хранение access/refresh-токенов;
- выбор одного доступного счётчика;
- автоматическая рекомендация целей, похожих на заявки и обращения, с возможностью изменить выбор;
- `/week` — последние 7 завершённых дней против предыдущих 7;
- объяснение общей динамики через потери и рост по источникам;
- отдельные списки страниц, где потеряли и где выросли;
- предупреждение, если вместо бизнес-целей выбраны вспомогательные клики;
- защита от ложных тревог на малых числах;
- автоматический отчёт по понедельникам в 09:00 МСК;
- `/disconnect` и `/delete_me` для удаления доступа и данных;
- никакие сырые данные Метрики и тексты отчётов не сохраняются.

Пример сообщения:

> **example.ru: что изменилось за неделю**
> Визитов стало на **280 меньше**: 840 ← 1 120 (−25%).
>
> Главное: **поиск** потерял 240 визитов, часть падения скрыли прямые заходы (+35).
>
> **Где потеряли**
> 🔻 Каталог: 210 ← 390 · −180 (−46%)
>
> **Что делать**
> 1. Проверить поиск: визиты снизились с 710 до 470.
> 2. Разобрать каталог: он потерял 180 визитов.

## Развернуть собственную копию

Этот раздел нужен разработчикам и компаниям, которые хотят хранить OAuth-токены только на своём сервере, изменить логику отчётов или использовать собственного Telegram-бота.

Требования: Docker Compose, Telegram-бот от [@BotFather](https://t.me/BotFather) и OAuth-приложение Яндекса.

1. Создайте приложение на [oauth.yandex.ru](https://oauth.yandex.ru/) типа «веб-сервисы».
2. Выдайте приложению только доступ `metrika:read`.
3. Укажите Redirect URI: `https://ваш-домен/oauth/callback`.
4. Скопируйте `.env.example` в `.env` и заполните значения.
5. Сгенерируйте ключ шифрования:

```bash
docker run --rm python:3.12-slim python -c "import base64,secrets; print(base64.urlsafe_b64encode(secrets.token_bytes(32)).decode())"
```

6. Запустите:

```bash
docker compose up -d --build
```

Сервис слушает `127.0.0.1:8080`. Перед ним нужен HTTPS reverse proxy. Пример для nginx лежит в [`deploy/nginx.conf.example`](deploy/nginx.conf.example).

## Локальная разработка

```bash
python3 -m venv .venv
.venv/bin/pip install -e '.[dev]'
set -a; source .env; set +a
.venv/bin/metrika-bot
```

Проверка:

```bash
.venv/bin/pytest -q
curl http://127.0.0.1:8080/health
```

## Команды

| Команда | Действие |
|---|---|
| `/start` | подключить Метрику или показать состояние |
| `/week` | собрать отчёт сейчас |
| `/counters` | выбрать сайт |
| `/goals` | выбрать заявки, звонки и продажи |
| `/pause` | отключить автоматический отчёт |
| `/resume` | включить его обратно |
| `/disconnect` | удалить OAuth-доступ |
| `/delete_me` | удалить все данные пользователя |
| `/privacy` | показать правила хранения данных |

## Как бот принимает решения

Бот не просит нейросеть придумывать причины. Он работает по проверяемым правилам:

- не тревожит из-за процентов на 2–4 визитах;
- считает изменение значимым только при достаточной базе, проценте и абсолютной разнице;
- показывает, какой источник упал и какие источники замаскировали падение ростом;
- разделяет страницы на потери и рост, а не смешивает их в одном рейтинге;
- отделяет падение целей при стабильном трафике от падения самого трафика;
- отмечает рост трафика без роста целей как возможную проблему качества.
- не называет клики по YouTube и другим вспомогательным элементам заявками.

Это первая версия. Пороги будут калиброваться на реальных обезличенных результатах, но данные пользователей не публикуются и не попадают в репозиторий.

## Безопасность

- OAuth state одноразовый и живёт 10 минут.
- Authorization Code защищён PKCE S256.
- Токены зашифрованы Fernet; ключ хранится только в окружении сервера.
- Приложению не нужен `metrika:write`.
- Секреты и SQLite исключены из Git.
- Перед публичным запуском используйте HTTPS и резервное копирование базы.

О найденных уязвимостях сообщайте приватно: порядок описан в [SECURITY.md](SECURITY.md).
